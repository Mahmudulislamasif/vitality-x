Introduction

Fitnofy is a Fitness Flutter UI Kit for gyms, fitness communities, and fitness platforms or services. This UI kit is perfect for gym owners and fitness business owners. Fitnofy has a wide range of pre-fixed features and options that can be easily modified with access to a large number of variations. It's a visually stunning and feature-filled UI kit that comes ready with tons of functionalities, removing the need for developing a fitness platform or app separately or from scratch. As a result, Fitnofy is a great tool for fitness entrepreneurs and business owners who want to manage an interconnected app platform for trainers, nutritionists, and regular users.

Features

User:

  • Can explore sessions and challenges (bundles of sessions focusing on a specific goal)
  • Can access two types of sessions (regular fitness training sessions and nutrition sessions)
  • Can follow trainers
  • Can make meal plans depending on thousands of dish options on the platform or trainersâ€™ recipes
  • Can manage the sleep schedule
  • Can access to a community
  • Has access to massive profile options

Trainer:

  • Can create/edit/manage challenges, sessions, and recipes
  • Have to publish their challenges/sessions for users to see
  • Can request admin to get featured
  • Can request admin to post an advertisement
  • Can have their challenges/sessions unpublished by the admin
  • Can request the admin to repost specific challenges/sessions
  • Can upload certifications
  • Has access to massive profile options

Admin:

  • Has access to a dashboard with insights
  • Manage trainers and their challenges/sessions
  • Can unpublish any challenges/sessions
  • Can verify a trainer by checking all their certifications and details
  • Can block/delete a trainer
  • Can manage the unpublished list
  • Can manage newly posted challenges/sessions
  • Can manage advertisements
  • Can manage advertisement request from trainers
  • Can manage republishing request
  • Can manage about the application
  • Has access to massive profile options

General features

  • All three panels have Dark and Light modes
  • All three panels support six (06) languages

Directory & files

This is a screenshot of the entire directory of fitnofy that contains all the necessary folders and files.

You can find the src folder and main.dart file under the lib folder.

Go to the src folder in order to get access to the following folders:

  • core
  • data
  • localization
  • models
  • modules
  • route
  • theme

The core folder contains the following folders:

  • constants (for constant variables)
  • reusableComponents (can be used in different parts of the app for different UI instances)
  • enums
  • utils

The data folder contains source data from both local and API and the dummy dataset on the app. Files in the models folder are used for releasing a set of information.

The modules folder can be divided into:

  • Controller (logical part of the app)
  • Page (defines the visual part of the app and contains the widget folder)
  • Binding (connecting the logical and visual parts)

The route folder contains:

  • App pages
  • App routes

pubspec.yaml(File) - Specification file (All the package and local assets (Like Images, Fonts, etc.) we use in this application is specify in this file.).

If you want to change the display name and bundle identifier follow the next steps

1 - Open info.plist file with Xcode and change the Bundle name with your application name

2 - Add the REVERSED_CLIENT_ID (you get it from GoogleService-Info) at the URL Schemes -> item 0
