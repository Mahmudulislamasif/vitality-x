Introduction

NFTO is a mobile app UI kit for NFT marketplaces. This theme is perfect for any kind of NFT creation and exchange marketplace. NFTO has a wide range of pre-fixed features and options that can be easily modified with access to a large number of variations. Its visually stunning and feature-filled theme comes ready with tons of functionalities, removing the need for developing a platform separately for front-end purposes.

Directory & files

This is a screenshot of the entire directory of NFTO that contains all the necessary folders and files.

You can find the src folder and main.dart file under the lib folder.

Go to the src folder in order to get access to the following folders:

  • core
  • data
  • localization
  • models
  • modules
  • route
  • service
  • theme

The core folder contains the following folders:

  • constants (for constant variables)
  • reusableComponents (can be used in different parts of the app for different UI instances)
  • utils

The data folder contains source data from both local and API and the dummy dataset on the app. Files in the models folder are used for releasing a set of information.

The modules folder can be divided into:

  • Controller (logical part of the app)
  • Page (defines the visual part of the app and contains the widget folder)
  • Binding (connecting the logical and visual parts)

The route folder contains:

  • App pages
  • App routes

pubspec.yaml(File) - Specification file (All the package and local assets (Like Images, Fonts, etc.) we use in this application is specify in this file.).

Display Name and Bundle identifier

If you want to change the display name and bundle identifier follow the next steps

1 - Open the project with Xcode then go to your /ios and open Runner.xcworkspace

2 - Change Display name and Bundle identifier in Targets > Runner change your application display and bundle identifier (the same bundle on GoogleServices-Info.plist ). then change the deployment info from 9.0 to IOS 11.0.

Configure the Info.plist

1 - Open info.plist file with Xcode and change the Bundle name with your application name

2 - Add the REVERSED_CLIENT_ID (you get it from GoogleService-Info) at the URL Schemes -> item 0

3 - Add your domain in the App Transport Security Settings key.
